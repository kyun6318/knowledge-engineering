## 3. 5대 리스크 및 완화 방안

| # | 리스크 | 위험도 | 발생 시나리오 | 비즈니스 영향 | 완화 방안 |
|---|--------|:---:|-------------|-------------|---------|
| R1 | **필터링 역전 (Top-K 부족)** | **High** | S&F가 100건만 넘기면 진짜 적합 후보가 잘림 | 매칭 품질 치명적 저하 | Top-K를 500~1,000건(10배)으로 Data Contract 명문화 |
| R2 | **S&F 산출물 지연** | **High** | W10~15 Batch 600K 에러/지연 → GraphRAG 개발 정지 | 프로젝트 일정 지연 | PubSub 자동 트리거 + Mock Data 선행 개발 + 주간 싱크 |
| R3 | **API 체인 레이턴시 초과** | **Medium** | 2-Tier 연속 호출로 사용자 체감 > 5초 | UX 치명적 저하 | SLA p95 < 3s 엄수, min-instances=1, 캐싱 |
| R4 | **Data Contract 스키마 충돌** | **Medium** | S&F가 필드명 변경 → Cypher 쿼리 실패 | 적재 파이프라인 장애 | JSON Schema Git 버저닝 + Phase별 Integration Test |
| R5 | **팀 분리 경계 모호** | **Medium** | 역할 혼동 → 중복/누락 | 책임 공백 | 73개 태스크 분류 테이블 + 주간 싱크 회의 |
