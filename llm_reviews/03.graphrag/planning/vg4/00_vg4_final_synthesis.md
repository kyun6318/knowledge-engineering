## 1. VG3 vs VC3 교차 비교

| 차원 | VG3 강점 | VC3 강점 | VG4 채택 |
|------|---------|---------|---------|
| **문서 구조** | ★ 4개 문서, 하향식(Top-Down) 읽기 흐름 | 6개 문서, 팀별 독립 참조 가능 | **하이브리드**: 통합 흐름 + 팀별 상세 분리 |
| **아키텍처 비전** | ★ 간결한 Polyglot + Loose Coupling 선언 | 분리 배경 3문제 + Polyglot 선언 (중복) | VG3 방식 (간결) |
| **역할 표현** | SLA 바인딩 + 핵심 미션 2줄 | ★ 8범주 상세 업무 테이블 + SLA 바인딩 | VC3 방식 (상세 + SLA) |
| **Data Contract** | JSON 1종(Candidate) + 필수 조건 3줄 | ★ JSON 3종(Candidate/Vacancy/Enrichment) + 필수 조건 + **PubSub 토픽 스키마** | VC3 방식 (3종 완비) |
| **산출물 명세** | 없음 (Data Contract에 산출물 암시만) | ★ 5종 산출물 테이블 (시점·형식·GCS경로·트리거) | VC3 방식 |
| **타임라인 시각화** | ★ Mermaid Gantt (간결) | ★ Mermaid Gantt (4번째 마일스톤 ① 추가) | VC3 방식 (4마일스톤) |
| **Work/Wait** | 서술형 (87% 언급) | ★ 정량 테이블 (구간별 순수 작업량) | VC3 방식 (정량) |
| **Go/No-Go** | 3개 마일스톤 (서술형) | ★ 3개 마일스톤 (테이블형 통과 조건) | VC3 방식 (테이블) |
| **Phase 상세** | Phase별 요약 (조건/작업/통과기준) | ★ Phase별 주차+DE/MLE 역할+산출물목록+선행작업 | VC3 방식 (주차 상세) |
| **S&F 상세** | 없음 (역할 테이블에 포함) | ★ 독립 문서 (6범주+v5 위치 매핑+품질메트릭+비용) | VC3 방식 (독립) |
| **비용** | 통합 1테이블 + 최적화 사유 칼럼 | ★ 통합 + **Phase별 세분화** (S&F/GraphRAG 각각) | VC3 방식 (세분화) |
| **리스크** | ★ "발생 시나리오" 칼럼 (구체적) | "비즈니스 영향" 칼럼 | **양쪽 병합**: 시나리오 + 비즈니스 영향 |
| **의사결정 포인트** | 없음 | ★ 11개 시점별 주체 재배치 | VC3 방식 |
| **Vector DB 기준** | 1줄 | ★ 3시나리오 테이블 | VC3 방식 |
