# 3. 팀별 R&R 및 비용 & 리스크 관리 (vg2)

> **목적**: 책임 소지를 명확히 분리하고, 전체 개발 및 운영 예산 내에서 각 팀이 효율적으로 자원을 운용할 수 있도록 규정한다.

---

## 3.1. S&F 팀 vs GraphRAG 팀 책임 명세서 (R&R)

### 3.1.1. S&F 팀
* **역할 책임**:
  * 원본 데이터베이스 쿼리를 통해 PII 마스킹을 "절대 누락 없이" 적용하는 것.
  * 정규 표현식 기반 파일 파서 관리 및 LLM Fallback 연계의 안정성.
  * 텍스트 임베딩 모델 선택(Vertex AI 등) 및 **1차 Vector 검색 엔진/API의 SLA (레이턴시 < 500ms) 달성**.
* **비용 예산**: 총액 **~$2,115~2,265** 내외 통제.
  * (주로 LLM 토큰 비용 $1,800대 및 Vector DB Cloud 인프라 비용).

### 3.1.2. GraphRAG 팀
* **역할 책임**:
  * S&F의 JSON 스키마를 신뢰하여, Graph 특화 속성 설계에만 시간 투자.
  * `[NEXT_CHAPTER]`나 `stage_match` 같은 직무 트렌드 상관관계를 Neo4j 그래프 알고리즘으로 모델링.
  * **2차 정밀 추천 체인의 SLA (Graph 메모리 서치 속도 < 2s) 보장**.
* **비용 예산**: 총액 **~$3,435~6,785** 내외 통제.
  * (주로 Neo4j Professional 라이선스 및 전문가(Human) Gold Label 적합도 평가 비용 $3,000 이상).

---

## 3.2. 총괄 비용 요약

전체 v5 비용 계획($5,527~$9,137)의 팀별 배분표:

| 항목 (Phase 통합) | S&F 팀 부담 (예산) | GraphRAG 팀 부담 (예산) |
|---|---|---|
| **LLM (Anthropic+Gemini)** | $1,807 (100%) | $0 |
| **Embedding Vector DB** | $52 (100%) | $0 |
| **Neo4j AuraDB** | $0 | $400 ~ $990 (100%) |
| **기타 Cloud 인프라** | $120 ~ $180 | $80 ~ $120 |
| **Gold Label (품질평가 인건비)**| $0 | $2,920 ~ $5,840 |
| **최종 합계** | **~$2,000 ~ $2,100** | **~$3,500 ~ $7,000** |

> **예산 최적화 포인트**: Vector 연산을 GraphDB에 얹지 않고 S&F가 외부 Milvus 등으로 덜어냄으로써, 상대적으로 고비용인 Neo4j의 Ram/Disk 사이즈를 **한 단계 축소(월 $50~$100 절감)**하여 전체 아키텍처 고도화 TCO 하락 효과를 누릴 수 있다.

## 3.3. 크리티컬 리스크 대장 및 완화(Mitigation) 플랜

아키텍처 스케일링을 위해 시스템을 나눔으로써 발생하는 필수 고려 리스크와 대응책을 정의한다.

| # | 식별된 리스크 | 위험도 | 영향 비즈니스 | 완화(Mitigation) 방안 (vg2) |
|---|---|:---:|---|---|
| **1** | **API 필터링 역전 (Top-K 부족)** | **High** | 사용자가 원하는 필드 요건에 하나도 맞지 않는 엉뚱한 매칭 도출 | S&F 1차 Recall 풀을 최종 목표(20건)의 **10배수 이상(500~1,000건)** 여유롭게 리턴하도록 시스템 제한 설정. |
| **2** | **S&F 산출물 병목 현상** | **High** | GraphRAG 팀 개발 대기(Idle) 증가 | GCS → **PubSub 자동 이벤트 트리거링 연동(I1)** 구축 및 단계별 Mock Data 제공으로 GraphRAG 개발 사전 완료 유도. |
| **3** | **API 체인 레이턴시 초과** | **Medium** | 에이전트 사용자 대기 시간 > 5초로 UX 치명상 | SLA p95 < 3s 엄수 (S&F 500ms + GraphRAG 2s). Neo4j의 Cold Start 대비용 **min-instances 최소 1대 유지**. |
| **4** | **조직 간 모델 매핑/스키마 충돌** | **Medium** | 속성명 변경으로 인한 Cypher 조회 실패 | JSON 스키마를 프로젝트 **단일 Data Contract 파일**로 버저닝 관리하고, Phase별 E2E Integration 파이프라인 CI 구축. |
| **5** | **Vector DB 선택 장기화** | **Low** | 검색 고도화 일정 지연 | Person 노드 < 1M 이면 Neo4j 기본 VectorIndex 활용 유지, **1M 초과 시** Milvus/Pinecone으로 즉각 독립(I2). |
