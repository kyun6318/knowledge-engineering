## 2. v3에서 잘 개선된 부분

### 2.1 에러 핸들링의 체계성

02 §8이 v3의 가장 큰 개선점. 에러 유형 6가지를 분류하고 각각의 정책(재시도 횟수, backoff, fallback, skip 기준)을 명확히 했다. dead-letter 큐 + 재처리 주기까지 실무적으로 유용한 설계.

### 2.2 상관 리스크 식별

05 §2.3에서 파싱 품질(P3)과 LLM 추출 품질(P1)이 **상관 리스크**임을 식별한 것은 v2에서 놓친 중요한 인사이트. Phase 0 PoC에서 "파싱+추출 통합 품질 측정"을 권장한 것도 적절하다.

### 2.3 운영 전략의 구체성

04의 롤백/재처리/증분 처리 섹션이 실질적으로 유용. 특히:
- Context JSON 버전 관리
- `prompt_version` 메타데이터
- 증분 처리의 비용 추정 (일 1,000건 × $0.00115 = $1.15/일)

### 2.4 비용 모델의 완전성

03에서 프롬프트 최적화 LLM 비용($200)과 오케스트레이션 인프라($600/년)를 추가한 것은 적절. 시나리오 A 총비용 $9,005는 현실적 추정.

## 3. v3에서 여전히 남아 있는 이슈

### 3.1 [Medium] ML Distillation 중복 — 부분 잔존

02 §9와 04 §2-3에 ML Distillation 내용이 여전히 일부 중복. 02에서는 "적용 범위(요약)"이라 명시했으나, 상세 실행 계획을 04에서만 다루는 원칙이 더 깔끔할 것. **실질적 영향은 미미**하므로 현재 수준으로도 충분히 수용 가능.

### 3.2 [Medium] NICE 감쇠 함수 검증 미포함

05 §2.4에서 NICE 데이터 매칭률 리스크를 다루지만, 02 §3.2의 `confidence = max(0.20, 0.60 - years_gap * 0.08)` 감쇠 함수의 **계수 검증 계획**은 여전히 없다. Phase 2 품질 평가 항목에 "NICE 감쇠 함수 적절성 확인"을 추가하면 좋겠음.

### 3.3 [Low] MappingFeatures r < 0.4 시 대응 계획

04 "핵심 의사결정 포인트"에 "MappingFeatures 조정 | human eval r값 | weight 재조정 / 피처 재설계"가 이미 포함되어 있다. 다만 r < 0.4 시의 구체 대응(예: 어떤 피처를 우선 재설계할지, 추가 기간 추정)은 없다. 이는 Phase 2 결과를 보고 판단하는 것이 합리적이므로, **현재 수준으로 충분**하다.

### 3.4 [Low] Hallucination 완화 전략 미반영

v2 리뷰 §5.2에서 지적한 LLM Hallucination 리스크(존재하지 않는 signal 생성, 정량 수치 날조)에 대한 완화 전략이 v3에 반영되지 않았다. 02의 프롬프트에 "근거 없이 추론하지 마세요" 지침이 있지만, 이는 프롬프트 레벨이므로 **후처리 레벨 검증**(evidence_span 존재 확인, confidence threshold 필터링)도 추가하면 좋겠음.

### 3.5 [Low] Phase 1에서의 프롬프트 최적화 반복 일정

타임라인에 "프롬프트 반복 최적화"가 별도로 드러나지 않는다. 다만 Phase 0 PoC(50건 테스트)에서 프롬프트 초안이 이미 검증/최적화되므로, Phase 1에서는 미세 조정만 필요. 14~18주의 범위가 이를 흡수할 수 있어 **현실적 문제는 아닐 것**이다.

## 5. 최종 판정

> v3 계획은 v2의 **핵심 부족 사항을 체계적으로 보강**했다. 특히 에러 핸들링, 운영 전략, 인력 배치, 타임라인 현실화는 "설계 문서"에서 "실행 가능한 계획"으로의 전환을 달성했다.
>
> 남아 있는 이슈는 모두 Medium~Low 수준으로, 계획 실행 과정에서 점진적으로 보완 가능하다.

### 개선 권장 사항 (우선순위순)

| 우선순위 | 항목 | 조치 |
|---|---|---|
| **Medium** | NICE 감쇠 함수 검증 | Phase 2 품질 평가 항목에 추가 |
| **Medium** | ML Distillation 중복 최소화 | 02에서 1문장 결론만, 04에서 상세 |
| **Low** | Hallucination 후처리 검증 | 02에 evidence_span 존재 확인 로직 추가 |
| **Low** | MappingFeatures r < 0.4 대응 | 현재 수준 충분, Phase 2 결과 후 구체화 |

### 전체 완성도

v3 계획은 **즉시 실행 가능한 수준**이다. Phase 0 PoC 결과에 따라 세부 조정이 필요하지만, 구조적으로 건전하고 리스크 관리가 충실하다. v2→v3에서 Critical/High 이슈가 모두 해결되었으며, 남은 이슈는 실행 과정에서 자연스럽게 보완할 수 있는 수준이다.
