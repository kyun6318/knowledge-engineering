## MEDIUM

### M1. compute_ranking_score() low_confidence_flag 필드명 미정의

- **위치**: `03.graphrag/separate/v8/graphrag/04_graphrag_g3_matching.md` L115~116
- **현상**:
  ```python
  "low_confidence_flag": sum(1 for f in ["stage", "vacancy", "domain", "culture", "role"]
                              if candidate.get(f"{f}_active", True)) <= 2
  ```
- **문제**: `candidate` dict에 `stage_active`, `vacancy_active` 등의 키가 Data Contract이나 정본 어디에도 정의되지 않음. 기본값 `True`로 인해 항상 5개 활성 → flag가 항상 `False`
- **권장**: 활성 피처 수를 별도 인자로 받거나, `FeatureResult.status == "ACTIVE"` 기반으로 산출하도록 로직 수정. 또는 pseudo-code 수준의 의도를 명시하는 주석 추가

### M2. 07_data_quality.md F1~F5 테이블 직접 기재 (레이어 경계)

- **현상**: F1 stage_match ~ F5 role_fit까지 데이터 품질 영향 테이블을 직접 기재

- **권장**: 테이블 상단에 `> 아래 테이블은 데이터 품질이 각 피처에 미치는 영향을 보여준다. 피처 정의는 01.ontology/v25/03_mapping_features.md 참조.` 교차 참조 주석 추가

### M3. SENIORITY_ORDER에 UNKNOWN 미포함 + 문서화 부재

- **위치**: `01.ontology/v25/03_mapping_features.md` L524~526
- **현상**: `SENIORITY_ORDER = {"JUNIOR": 1, "MID": 2, "SENIOR": 3, "LEAD": 4, "HEAD": 5}` — UNKNOWN 없음
