# 4. 조직 간 비용 및 리스크 방어 플랜 (VG3)

> 독립적인 마이크로서비스로 나뉘었다 하더라도 최종 산출물은 통합되어야 하므로, 각 팀의 책임 비용(Cost Center)을 정확히 산정하고 이음새 구간의 리스크 대응책을 미리 선언한다.

---

## 4.1 전체 비용(TCO) 산정 및 분할 테이블

v5 원본 문서에서 추정한 전체 개발/운영 비용($5,527~$9,137)을 각 단위 기능별로 각 팀 예산에 분할 할당한다.

| 사용 인프라/API | S&F 팀 부담 (비용 센터) | GraphRAG 팀 부담 (비용 센터) | 최적화 사유 |
|---|---|---|---|
| **LLM (Anthropic / Gemini)** | $1,807 (100%) | $0 | 하드 정보와 임베딩 맥락을 모두 S&F팀이 선-파싱. |
| **Embedding (Vertex AI)** | $52 (100%) | $0 | - |
| **Neo4j AuraDB 라이선스** | $0 | $400 ~ $990 (100%) | Vector 적재 짐을 덜어내어, AuraDB 스케일을 한 단계 Downgrade (-$100/mo 효과) 유도. |
| **빅쿼리/서빙(Cloud Run/GCS)** | $120 ~ $180 | $80 ~ $120 | 이벤트(PubSub) 기반이므로 API 서버가 유휴 시 Zero Scale 됨 (비용 감소). |
| **Gold Label (품질 평가용)** | $0 | $2,920 ~ $5,840 | 최종 모델 타당성은 GraphRAG 팀에서 휴먼 인적 자원으로 검증. |
| **합계 (팀별 추정 예산안)** | **~$2,000 ~ $2,100** | **~$3,500 ~ $7,000** | - |

## 4.2 필수 선언 5대 리스크 (Risks & Mitigation)

| ID | 식별된 치명적 리스크 | 발생 시나리오 | 구체적 방비(Mitigation) 플랜 (VG3 제언) |
|---|--|---|---|
| **R1** | **에이전트 연계 레이턴시 초과** | 두개(S&F, G-RAG)의 API를 연속 호출하며 사용자 체감 응답시간이 5초 이상 폭증함 | 2-Tier API SLA 규약: S&F 하위 쿼리를 무조건 **< 500ms** 방어, Neo4j in-list 쿼리를 **< 2s** 엄수. Cloud Run min-instances 활성화. |
| **R2** | **"필터링 역전" (Top-K 고갈)** | S&F가 넘겨준 100건 안에 진짜 적합한 후보가 이미 잘려서 안 넘어옴 | S&F가 1차 검색 시 최종 필요 후보의 10배수 이상인 **500 \~ 1,000건**의 Pool을 폭넓게 던지도록 Data Contract 명문화. |
| **R3** | **S&F 파이프라인 지연** | W10~15 구간 대량 Batch 600K 처리가 에러나 트래픽으로 지연되어 GraphRAG 팀이 "개발 정지" 당함 | Event-driven (GCS ➔ PubSub) 자동화로 S&F Chunk 완료 즉시 자동 E2E 적재. 지연되는 동안 GraphRAG는 선행 API 모의 대응 처리 (Utilization 하락 방지). |
| **R4** | **Data Contract 필드 불일치 충돌** | S&F가 텍스트를 추출하다 JSON Schema 필드명을 일방적으로 변경, Graph의 프로퍼티 연산 장애 | 팀 E2E 통합 시작 전 Git 기반의 JSON Schema(JSONL Validator)를 상호 결재된 공용 버저닝 Repository로 승격하여 E2E 깨짐을 사전 Validation. |
| **R5** | **Vector DB 선택 장기화 표류** | S&F 팀이 벡터를 Neo4j에 넣을지 별도 DB(Pinecone 등)에 할지 결정을 끌며 연계 지연 유발 | **Person 엔티티 < 1M 기준**으로 조건부 합의 완료. (규모 작으면 Neo4j의 Vector Index 단일 사용, 초과 시 별도 독립 채택). |
