# VC4 — VG3 vs VC3 비교 리뷰 및 합성 전략

> **핵심 방향성 (불변)**: 하드필터·PII·파싱·임베딩은 **S&F 팀**이, `[NEXT_CHAPTER]` 기반 그래프 관계·매칭은 **GraphRAG 팀**이 전담한다.

---

## 1. VG3 vs VC3 교차 비교

| 차원 | VG3 강점 | VC3 강점 | VC4 채택 |
|------|---------|---------|---------|
| **문서 구조** | ★ 4개 문서, 하향식 읽기 흐름 | 6개 문서, 팀별 독립 참조 가능 | **VC3 기반** (팀별 독립 문서 유지하되 하향식 읽기 순서 명시) |
| **아키텍처 비전** | ★ 간결한 Polyglot + Loose Coupling (2줄) | 분리 배경 3문제 + Polyglot (약간 중복) | VG3 방식 (간결) |
| **역할 테이블** | 핵심 미션 + SLA + 인프라 (4칼럼 1테이블) | ★ 8범주 상세 업무 테이블 + SLA 별도 인용 | VC3 방식 (상세) |
| **Data Contract** | JSON 1종(Candidate) + 필수조건 3줄 | ★ JSON 3종(Candidate/Vacancy/Enrichment) + PubSub 토픽 스키마 | VC3 방식 (3종 완비) |
| **산출물 명세** | 없음 | ★ 5종 테이블 (시점·형식·GCS 경로·트리거) | VC3 방식 |
| **타임라인** | ★ Mermaid Gantt (깔끔, 3마일스톤) | ★ Mermaid Gantt (4인터페이스 마일스톤, PoC ① 추가) | VC3 방식 (4마일스톤) |
| **Work/Wait** | 서술형 요약 (87%) | ★ 정량 테이블 (구간별 순수 작업량, 13주/1.5주 유휴) | VC3 방식 (정량) |
| **Go/No-Go** | 3개 서술형 | ★ 3개 테이블형 통과 조건 | VC3 방식 |
| **Phase 상세** | Phase별 주요 작업+조건+통과기준 (요약) | ★ Phase별 주차+DE/MLE 역할+산출물 목록+선행작업 상세 | VC3 방식 (주차 상세) |
| **S&F 상세** | 없음 (역할 테이블에만 포함) | ★ 독립 문서 (6범주+v5 위치 매핑+품질 메트릭+비용) | VC3 방식 (독립) |
| **비용** | 통합 1테이블 + "최적화 사유" 칼럼 | ★ 통합 + Phase별 세분화 (S&F/GraphRAG 각각) | VC3 방식 (세분화) |
| **리스크** | ★ "발생 시나리오" 칼럼 (구체적) | "비즈니스 영향" 칼럼 (경영진 보고형) | **둘 다 채택**: 시나리오 + 비즈니스 영향 병합 |
| **의사결정 포인트** | 없음 | ★ 11개 시점별 주체 재배치 | VC3 방식 |
| **Vector DB 기준** | 1줄 | ★ 3시나리오 테이블 | VC3 방식 |

## 2. VC4 합성 원칙

| # | VG3에서 가져옴 | VC3에서 가져옴 |
|---|--------------|--------------|
| 1 | 간결한 Polyglot 선언 (분리 배경을 짧고 명확하게) | 8범주 S&F 상세 역할 + SLA 바인딩 |
| 2 | — | JSON 3종 + PubSub 토픽 스키마 + 산출물 5종 |
| 3 | — | Mermaid Gantt (4마일스톤) + Work/Wait 정량 |
| 4 | — | Phase별 세분화 비용 (양 팀 각각) |
| 5 | "발생 시나리오" 칼럼 (리스크 구체화) | "비즈니스 영향" 칼럼 (경영진 보고) |
| 6 | — | 11개 의사결정 포인트 + Go/No-Go 테이블 |

> **결론**: VC3가 이미 대부분의 강점을 보유한 완성도 높은 문서. VG3에서 추가 보강할 점은 **① 분리 배경의 간결화** 와 **② 리스크에 "발생 시나리오" 칼럼 추가** 두 가지뿐.
