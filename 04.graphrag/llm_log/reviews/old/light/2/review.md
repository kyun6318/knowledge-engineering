# GraphRAG light.2 실행 계획 리뷰

> **리뷰 대상**: light.2 계획 (6개 문서) — light.1 리뷰 피드백 반영본
> **리뷰 초점**: 실현 가능성, 과도한 일정/기능, 부족한 일정/기능
> **리뷰 일자**: 2026-03-08

---

## 1. 총평

light.2는 light.1 리뷰에서 지적한 **핵심 문제 5개를 모두 반영**한 현실적 보정본이다.
Pre-Phase 0 명시, Phase 1-C 확대, 80% 가용률 시나리오, Batch API 3-시나리오, Cloud Workflows 후속 이관 등 주요 권장사항이 적절히 수용되었다.

**전체 실현 가능성 판정: 높음 (75~85%)**

light.1(60~70%) 대비 실현 가능성이 유의미하게 향상되었으며, 남은 리스크는 대부분 **외부 의존성**(법무, Batch API quota, 도메인 전문가 확보)에 집중되어 있다.

## 2. 실현 가능성 분석

### 2.1 높은 실현 가능성 (Green)

| 항목 | 근거 |
|------|------|
| Pre-Phase 0 명시적 포함 [light.2-1] | light.1의 가장 큰 일정 리스크가 해소. 이해관계자 기대 관리에 필수적 |
| Phase 1-C 2주 확대 [light.2-2] | Org ER 한국어 회사명 특화 + 모듈 개발 시간이 현실적으로 반영 |
| 프롬프트 튜닝 전용 0.5주 [light.2-3] | 4개 프롬프트 × 3~5회 반복 = 15~20 라운드를 명시적으로 계산한 점이 우수 |
| Phase 1-D 1주 확정 [light.2-5] | 테스트 코드 = 구현 코드 시간이라는 현실 반영 |
| Cloud Workflows 후속 이관 [light.2-6] | 복잡도 감소 + 0.5주 절약. Phase 2도 Makefile로 충분한 근거가 타당 |
| Gold Test Set 선행 시작 [light.2-8] | Phase 1 후반 분산 라벨링으로 Phase 2 과부하 해소. 0.5주 절약 |
| Dead-letter 별도 0.5주 [light.2-10] | 13,500~22,500건 규모의 재처리를 체계적으로 다룸. 분류 로직이 구체적 |
| Batch API 3-시나리오 [light.2-7] | 낙관/기본/비관 제시로 이해관계자의 기대 범위가 명확해짐 |

## 9. 최종 판정

> **light.2 계획은 light.1 리뷰의 핵심 피드백을 충실히 반영한 현실적 계획이다.**
>
> - **실현 가능성**: 75~85% (light.1의 60~70%에서 상승)
> - **타임라인 신뢰도**: 공식 일정 18.5~20.5주(Pre 포함)는 100% 풀타임 기준 합리적
> - **비용 투명성**: Gold Label 산출 근거, 환율, contingency가 모두 명시되어 예산 정당성 확보
