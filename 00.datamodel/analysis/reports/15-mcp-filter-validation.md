## 1. 분석 개요

### 1.1 목적
MCP(Model Context Protocol)를 통해 에이전트가 이력서 검색 API를 호출할 때, 각 Array/Enum 필터 필드의 코드값이 통계적으로 충분한 볼륨을 보유하는지 검증한다. 이를 통해 에이전트에게 "유효 필터값 사전(dictionary)"을 제공하여 빈 결과(empty result) 및 과다 결과(overwhelming result) 문제를 사전 방지한다.

## 7. 에이전트 필터링 전략 권고

### 7.1 필수 기본 필터 (항상 적용)

```
visibility_type = 'PUBLIC'
```

### 7.2 권장 필터 조합 패턴

| 패턴 | 필터 조합 | 예상 결과 범위 | 용도 |
|------|----------|--------------|------|
| **경력자 직무 탐색** | career_type + jcc + county(광역) | 50K~200K | 가장 일반적 |
| **경력자 직무 + Freshness** | career_type + jcc + county + updated_at 90d | 10K~50K | 활성 후보 |
| **적극 구직자** | job_search_status=ACTIVE + jcc | 10K~50K | 높은 응답률 기대 |
| **신입 탐색** | career_type=NEW_COMER + education_level + county | 50K~200K | 신입 채용 |
| **특정 산업** | career_type + industry_codes + county | 20K~100K | 산업 특화 |

### 7.3 필터 추가 순서 가이드

에이전트가 결과 수를 조절할 때 필터 추가 우선순위:

1. **career_type** (필수) - 경력/신입 구분
2. **job_classification_codes** (1순위) - 직무 특정. 입력률 82.6%
3. **county_codes** (2순위) - 지역 한정. 입력률 87.0%. 광역 코드 사용
4. **updated_at** (3순위) - Freshness. 90일 기준 시 약 60~70% 축소
5. **job_search_status** (4순위) - ACTIVE 필터 시 3.7%로 급격 축소
6. **employment_types** (5순위) - 고용형태 세분화
7. **final_education_level** (6순위) - 학력 필터
8. **industry_codes** (보조) - 입력률 34%. 결과 과다 시 보조 필터로만
9. **job_keyword_codes** (보조) - 입력률 48.4%. 세부 키워드 매칭 시만

### 7.4 빈 결과 방지 규칙

1. **3-필터 이하 조합 시작**: 기본(visibility) + career_type + jcc 로 시작
2. **결과 < 1,000건 시 필터 완화**: county 범위 확장 또는 freshness 기간 연장
3. **결과 > 500,000건 시 필터 추가**: county, freshness, education_level 순으로 추가
4. **county_codes는 광역 코드만 사용**: 구/군 단위 코드는 볼륨 부족 위험
5. **industry_codes 단독 사용 금지**: 입력률 34%로 66% 이력서 누락

## 8. 한계 및 주의사항

[LIMITATION] 다음 제약사항을 인지해야 한다:

1. **코드값 의미 미매핑**: 본 분석의 코드값(4010000131 등)은 코드허브 매핑 없이 숫자 코드만 제시. 실제 에이전트 사전 구축 시 코드허브 API를 통한 한글 라벨 매핑이 필수적이다.

2. **입력률 편향**: industry_codes(34%), job_keyword_codes(48.4%)는 절반 이상의 이력서에 값이 없다. 이 필드 기반 필터링은 구조적으로 모수 손실을 야기한다.

3. **시점 한계**: 본 분석은 2026-03-04 기준 스냅샷이다. 이력서 수는 지속 증가하며, 코드별 분포도 계절/경기에 따라 변동한다.

4. **필터 간 독립성 가정**: 필터 조합 시뮬레이션은 실측값이나, 코드값 간 상관관계(예: 특정 직무-지역 쏠림)는 모든 조합에 대해 검증되지 않았다.

5. **MCP API 실제 동작 차이**: ClickHouse 직접 쿼리와 MCP API의 필터 처리 로직(hasAny vs hasAll, OR vs AND 등)이 다를 수 있다.

6. **county_codes 코드 체계**: ASKR 접두사 기반 코드 체계이며, 지역명 매핑은 Report 08b(codehub `foreign_attributes[JOBKOREA].display_name`) 기준으로 정정 완료.
