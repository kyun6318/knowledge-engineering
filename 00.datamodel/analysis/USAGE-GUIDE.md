## 1. Executive Summary — 의사결정자를 위한 핵심 요약

### 한 줄 요약

> 802만 이력서 중 **즉시 서비스에 활용 가능한 고신뢰 풀은 약 29만 명**(ACTIVE 구직자)이며, 단계적 확장을 통해 778만 전체 사용자까지 커버리지를 넓힐 수 있다.

### 5가지 핵심 사실

| # | 발견 | 서비스 영향 | 즉시 조치 |
|---|------|-----------|----------|
| 1 | **데이터 무결성은 완전하다** — 14개 테이블 간 고아 레코드 0건 | 테이블 간 JOIN이 안전하여 복합 검색·추천 로직 구현 가능 | 추가 정합성 작업 없이 서비스 적용 착수 |
| 2 | **이력서 31.6%가 5년 이상 방치** — 90일 이내 활성은 13.9%에 불과 | 오래된 이력서가 검색 결과에 노이즈로 작용, 응답률 저하 원인 | 신선도 기반 랭킹 가중치 즉시 도입 |
| 3 | **ACTIVE 구직자(29만 명)의 응답 가능성이 3.85배 높다** | 소싱·아웃리치 시 ACTIVE 우선 타겟팅으로 전환률 극대화 | CRM 대상 풀을 ACTIVE 중심으로 재설정 |
| 4 | **경력자 vs 신입의 이력서 품질 격차가 극심하다** (74.6점 vs 38.2점) | 신입 이력서는 정보 부족으로 매칭 정확도 저하 | 신입 대상 이력서 완성도 코칭(nudging) 도입 |
| 5 | **스킬 생태계가 급변 중** — AI 관련 스킬 3~7배 성장 | 기존 스킬 분류 체계로는 최신 인재 검색 불가 | 스킬 코드 정규화 파이프라인 구축 착수 |

## 3. 서비스별 활용 로드맵

### 3.1 검색/필터 API — 1순위

**현황**: 직무(82.6% 입력) · 지역(87.0% 입력) · 고용형태(80.2% 입력) 필터는 즉시 활용 가능. 산업(34.0%)은 보완 필요.

| 단계 | 적용 내용 | 데이터 근거 | 성공 기준 |
|------|----------|-----------|----------|
| **Phase 1** | 직무코드 TOP 100 인덱싱 (84~86% 커버) + 지역코드 전체 | Report 13a, 13b | 빈 결과율 < 5% |
| **Phase 2** | 신선도 필터 추가 (90일/1년/3년 구간) | Report 12, 07b | 과다 결과율 < 10% |
| **Phase 3** | 스킬 기반 검색 (codehub 매핑 스킬 우선) | Report 06, 16 | 검색 정밀도 향상 |
| **Phase 4** | 산업코드 63개 전체 인덱싱 + 자격증 TOP 100 | Report 13a | 필터 커버리지 95%+ |

**필터 설계 임계치** (Report 15 기준):

| 필터 | 최소 결과 보장 임계치 | 근거 |
|------|---------------------|------|
| 직무코드 | 단일 코드 1,000건+ | TOP 100 기준 |
| 지역코드 | 단일 코드 10,000건+ | 시·도 단위 |
| 산업코드 | 단일 코드 5,000건+ | 중분류 기준 |
| 스킬코드 | 단일 코드 500건+ | 고빈도 스킬 한정 |

**가드레일**:
- 결과 0건 시: 가장 좁은 필터부터 순차 완화 (스킬 → 산업 → 신선도 → 지역)
- 결과 500,000건+ 시: 신선도·ACTIVE 필터 자동 추가

### 3.2 랭킹/추천 — 2순위

**현황**: 현재 랭킹에 이력서 품질·신선도가 반영되지 않아, 5년 전 이력서가 상위 노출될 수 있음.

| 단계 | 적용 내용 | 데이터 근거 | 성공 기준 |
|------|----------|-----------|----------|
| **Phase 1** | 신선도 가중치 도입 (반감기 31.5개월 적용) | Report 12, 18 | 상위 50건 중 90일 이내 비율 > 30% |
| **Phase 2** | 품질 점수 보조 신호 (섹션 보유 기반) | Report 11, 18 | 상위 노출 후보의 평균 품질 점수 향상 |
| **Phase 3** | ACTIVE 부스트 (구직 의사 확인자 우선) | Report 07b, 12 | 열람 후 연락 전환률 개선 |
| **Phase 4** | 스킬 유사도 기반 재랭킹 | Report 06, 16 | 검색 키워드 대비 매칭 정밀도 향상 |

**랭킹 스코어 후보 공식** (AI 추정):
```
score = relevance_base
      × freshness_decay(반감기=31.5개월)
      × quality_boost(섹션 보유율 기반)
      × active_boost(ACTIVE=1.5x, 90일이내=1.2x, 기타=1.0x)
```

### 3.3 CRM / 아웃리치 — 3순위

**현황**: ACTIVE 구직자 29만 명 중 90일 이내 갱신자의 응답 가능성이 3.85배 높음.

| 단계 | 적용 내용 | 데이터 근거 | 성공 기준 |
|------|----------|-----------|----------|
| **Phase 1** | ACTIVE + 90일 이내 갱신군 우선 타겟팅 | Report 07b, 12 | 연락 성공률 향상 |
| **Phase 2** | 경력자/신입 세그먼트 분리 메시징 | Report 14, 17 | 세그먼트별 전환률 차이 확인 |
| **Phase 3** | 스킬 트렌드 기반 맞춤 포지션 추천 | Report 16 | 후보자 관심도(클릭률) 향상 |

**타겟 세그먼트 우선순위**:

| 세그먼트 | 규모 | 특성 | 아웃리치 전략 |
|---------|------|------|-------------|
| ACTIVE + 90일 이내 | ~13.8만 명 | 가장 높은 응답 가능성 | 즉시 연락, 포지션 매칭 |
| ACTIVE + 90일~1년 | ~10.5만 명 | 구직 의사 있으나 이력서 갱신 필요 | 이력서 업데이트 유도 후 연락 |
| 비ACTIVE + 90일 이내 | ~80만 명 | 최근 갱신했으나 구직 상태 미설정 | 구직 상태 확인 넛지 |
| 경력자 + 5년+ 미갱신 | ~175만 명 | 잠재 이직 대상 | 장기 nurturing 캠페인 |

### 3.4 이력서 완성도 코칭 — 4순위

**현황**: 평균 품질 52.4점, 신입 38.2점. 주요 미입력 섹션이 검색·매칭 정확도를 저하시킴.

| 미입력 섹션 | 미입력률 | 개선 시 영향 | 넛지 전략 |
|-----------|---------|------------|----------|
| **스킬** | 36.2% (미보유) | 스킬 기반 검색 커버리지 확대 | "00 직무의 인기 스킬 N개를 추가해보세요" |
| **어학** | 93.7% (미보유) | 글로벌 포지션 매칭 가능 | 어학 시험 점수 간편 입력 UI 제공 |
| **경력기술서** | 83.1% (미보유) | 경력 내용 기반 정밀 매칭 | 경력 항목 선택 시 자동 기술서 초안 제안 |
| **자격증** | 46.0% (미보유) | 자격 요건 필터 매칭 | 직무별 추천 자격증 가이드 |
| **자기소개서** | 35.9% (미보유) | 지원 동기 매칭, 문화적합도 판단 | 템플릿 기반 작성 가이드 |

## 5. 단계별 적용 체크리스트

### Phase 1: 즉시 적용 (0~2주)

- [ ] 검색 API에 `main_flag=1` + `visibility=PUBLIC` 기본 필터 적용
- [ ] 직무코드 TOP 100 기반 인덱싱 구현 (Report 13a)
- [ ] 지역코드 전체 인덱싱 (63개, Report 13a)
- [ ] ACTIVE 구직자 우선 노출 로직 추가
- [ ] 빈 결과 시 필터 순차 완화 로직 구현

### Phase 2: 신선도·품질 반영 (2~6주)

- [ ] 신선도 감쇠 함수 도입 — `user_updated_at` 기준, 반감기 31.5개월 (Report 12, 18)
- [ ] 이력서 품질 점수 계산 파이프라인 구축 (Report 11, 18)
- [ ] 랭킹에 신선도·품질·ACTIVE 가중치 적용 (Report 18 모델 참조)
- [ ] CRM 대상 풀을 ACTIVE + 90일 이내로 재설정

### Phase 3: 스킬·매칭 고도화 (1~3개월)

- [ ] 스킬 코드 정규화 파이프라인 구축 (codehub 매핑 확장)

- [ ] 경력×학력 교차 매칭 로직 (Report 17)
- [ ] 이력서 완성도 코칭 nudging 시스템 구축 (Report 07)

### Phase 4: 검증·최적화 (지속)

- [ ] A/B 테스트로 각 Phase 효과 검증
- [ ] 가드레일 모니터링 (빈결과율, 과다결과율, 응답률)

### 롤백 조건

서비스 적용 후 아래 조건 발생 시 즉시 이전 버전으로 롤백:

- 빈 결과율이 기존 대비 **2배 이상** 증가
- 과다 결과(500K+) 비율이 **20% 이상** 증가
- 연락 응답률이 기존 대비 **30% 이상** 하락
- 사용자 불만(CS) 건수가 기존 대비 **50% 이상** 증가
