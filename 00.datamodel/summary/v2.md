> 작성일: 2026-03-10 | 분석 기준일: 2026-02-27
> v2 대비 변경: 매핑 테이블(Designation→Seniority, Position→Scope) 추가, 비교 방법별 Confidence 보정 테이블 추가, Resume 품질 등급 분포 및 서비스 가용 풀 추가, Operating Model Facets 구조화 시그널 추가, Domain Depth 구조화 추출 로직 추가

## 5. 엣지 구현 데이터 품질

### 5.1 FK 무결성

14개 테이블 간 **15개 FK 관계에서 고아 레코드 0건** — 데이터 무결성 완전.

### 5.2 JOIN 전략

| JOIN 유형 | 관계 | 근거 |
|----------|------|------|
| **INNER JOIN** | resume ↔ profile (siteUserMappingId) | 양방향 완전 매핑 |
| **INNER JOIN** | resume ↔ workcondition (resumeId) | 완전 1:1 |
| **LEFT JOIN** | resume → career, education, certificate, skill 등 | 선택적 (6.3%~95.6%) |

**주의**: 실제 JOIN 키는 `siteUserMappingId` (명세의 `siteUserRef`가 아님)

### 5.3 엣지별 데이터 가용성

| 엣지 | 데이터 소스 | 가용성 | 비고 |
|------|-----------|--------|------|
| `Person-[:HAS_CHAPTER]->Chapter` | resume → career | 68.9% (5.52M) | LEFT JOIN |
| `Chapter-[:NEXT_CHAPTER]->Chapter` | career.started_on 정렬 | 68.9% | 동일 resume 내 시간순 |
| `Chapter-[:PERFORMED_ROLE]->Role` | career.jobClassificationCodes | ~100% (경력 보유자) | codehub 100% 매핑 |
| `Chapter-[:USED_SKILL]->Skill` | resume.skill | 38.3% | **경력-스킬 직접 연결 불가** (resume 단위) |
| `Chapter-[:OCCURRED_AT]->Organization` | career.companyName | 99.96% | 정규화 후 매핑 |
| `Chapter-[:PRODUCED_OUTCOME]->Outcome` | careerDescription + selfIntroduction | 16.9%~64.1% | LLM 추출 |
| `Chapter-[:HAS_SIGNAL]->SituationalSignal` | 텍스트 소스 | ~65-70% | LLM 추출 |
| `Organization-[:HAS_VACANCY]->Vacancy` | job-hub | 별도 분석 필요 | |
| `Vacancy-[:REQUIRES_ROLE]->Role` | overview.job_classification_codes | 별도 분석 필요 | codehub 직접 매핑 가능 |

**핵심 제약**: `careerDescription`은 `career_id` FK 없음 → career 항목별 매핑 불가, resume 단위 귀속. Outcome/Signal 추출 시 LLM이 텍스트 컨텍스트로 판단 필요.
